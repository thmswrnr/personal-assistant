# YouTube

Two capabilities:
- **Transcript** of any public video (no account needed) — to summarize or answer.
- **Your account** (read-only): subscriptions and a "what's new" feed of recent uploads
  from channels you follow.

## Transcript (run via bash)

```bash
# Accepts a full URL or a bare 11-char video id; optional --lang (default en)
node /app/.pi/skills/google/youtube/scripts/transcript.mjs "https://www.youtube.com/watch?v=VIDEO_ID"
node /app/.pi/skills/google/youtube/scripts/transcript.mjs "VIDEO_ID" --lang de
```

Prints `{id, title, uploader, durationSeconds, url, lang, autoGenerated, chars, truncated, transcript}`.

- Summarize: one-line gist, then key points as bullets. Keep it tight unless asked for detail.
- If `autoGenerated` is true, captions are machine-made (minor errors, no punctuation) —
  summarize the meaning, don't quote as exact. If `truncated`, say so and summarize what you have.
- If it errors with "no captions available", say so plainly; try `--lang` for other languages.

## Your subscriptions & new videos (run via bash)

```bash
# Channels the user is subscribed to
node /app/.pi/skills/google/youtube/scripts/feed.mjs subscriptions

# Recent uploads across all subscriptions, last N days (default 7)
node /app/.pi/skills/google/youtube/scripts/feed.mjs feed 7
```

- `subscriptions` prints `{count, subscriptions:[{channel, channelId}]}`.
- `feed` prints `{sinceDays, channelsChecked, newVideos, videos:[{title, channel, published, url}]}`,
  newest first. Summarize the highlights grouped by channel or topic — don't dump all 40.
  Offer to fetch a transcript of any the user wants summarized (chain into `transcript.mjs`).

## Rules

- **Don't invent** videos, channels, or content — only report what the commands return.
- If the account commands error (e.g. missing `youtube.readonly` scope), say so plainly;
  the user may need to re-run `scripts/google-oauth.mjs`. The transcript command needs no account.
